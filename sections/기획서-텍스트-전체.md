### 섹션 2. 사전알림 버튼
**기획 의도**
- 사전 참여 유도 → 페스타 시작 시 리텐션 확보

**상세 스펙**
- "🔔 사전 알림 받기" 버튼 1개
- 클릭 시 → 버튼 영역 사라지고 쿠폰팩 카드 노출
- 알림 동의 처리 필요 (앱 푸시)

**개발 핵심 필요사항**
- [FE] 버튼 클릭 → 쿠폰팩 카드 전환 인터랙션
- [서버] 사전알림 동의 API
- [서버] 쿠폰팩 발급 API (3종 일괄)

### 섹션 3. 쿠폰팩 카드
**기획 의도**
- 페스타 기간(6/11~7/19)에 사용 가능한 쿠폰 사전 지급

**상세 스펙**

- 쿠폰팩 수령 완료 시 "페스타 시작 시 알림을 보내드릴게요" 안내

**개발 핵심 필요사항**
- [서버] 쿠폰 종류/금액/기간 어드민에서 설정 가능하도록
- [사업] 쿠폰 예산 확인 필요 (더하기 쿠폰 예산)

### 섹션 1. KV 영역

**개발 핵심 필요사항**

---

### 섹션 2. 먹을복 결과 카드
**기획 의도**
- CTA 없이 입장 즉시 결과 노출 (전환율 개선)
- 최근 12주 주문 내역 기반 맛 유형 분석

**상세 스펙**
- 맛 유형: 매운맛 / 짠맛 / 단맛 / 건강한맛
- 각 맛별 % 바 그래프
- 한줄평 텍스트 (데이터 기반 자동 생성)
- 공유 버튼 (카카오톡)

**개발 핵심 필요사항**
- [데이터] 12주 주문 내역 → 맛 유형 분류 로직
- [데이터] 한줄평 텍스트 생성 로직
- [FE] 결과 카드 공유 기능 (카카오 SDK)

---

### 섹션 3. 맛별 가게 추천
**기획 의도**
- 결과 기반 개인화 가게 추천 → 주문 전환
- 브랜드가 아닌 가까운 지점 노출

**상세 스펙**
- 맛 탭 4개 (매운맛/짠맛/단맛/건강한맛)
- MY 탭 = 유저의 1순위 맛 유형
- 가게 정렬: 최근 주문이력 > 영업중 > 별점 > 주문수
- 브랜드 아코디언 → 지점 리스트
- 새로고침 버튼

**개발 핵심 필요사항**
- [데이터] 맛 유형별 가게 매핑 로직
- [데이터] 가게 스코어링 (주문이력 > 영업중 > 별점 > 주문수)
- [FE] 위치 기반 가까운 지점 노출
- [사업] 브랜드 섭외 (주차별 브랜드/메뉴 정리 필요)

---

### 섹션 4. Backfill - 주문 0회 유저 (별도 파일: section1-w12-backfill-noorder.html)

### 섹션 2. 배민 추천 꿀조합 (페어딜)

- [서버] 조합별 쿠폰 발급
- [사업] 브랜드 섭외 + 쿠폰 예산
